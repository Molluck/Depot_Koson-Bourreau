Importance du module dans l'enseignement

 

Objectif

  • Savoir prorammer un client « riche » selon le motif d'architecture logicielle (design pattern) reposant sur la séparation des 3 éléments suivants :

    • les données (HTML5)
    • la présentation (CSS3)
    • le comportement (JS)

    ► M    odèle MVC (Model-View-Controller) vu en cours de Java

Prérequis

  • M2105 : Introduction aux interfaces homme-machine (IHM)
  • M3104 : Programmation Web côté serveur
  • M3105 : Conception et programmation objet avancées

 

Contenu

  • Modèle de documents Web, en anglais DOM (Document Object Model)
  • Gestion dynamique du DOM avec JavaScript (JS) et autres cadriciels (frameworks) basés sur JS (jQuery)
  • Programmation événementielle
  • Requêtes synchrones et asynchrones
  • Formats d'échange de données (XML, JSON)

 

Cours

  • Cours 1 : Introduction
  • Cours 2 : Javascript
  • Cours 3 : Requêtes asynchrones et formats d'échange de données
  • Cours 4 : Exemples de bibliothèques et d'autres langages

 


Travaux

5 travaux dirigés étalés sur 7 séances :

  • TD 1 : Introduction à JavaScript et au DOM. (séance 1 et séance 2)
  • TD 2 : Poursuite de l'apprentissage du JavaScript et du DOM. (séance 3)
  • TD 3 : Mise en pratique du JavaScript sur des applications plus concrètes. (séance 4)
  • TD 4 : Étude des mécanismes asynchrones (AJAX) et présentation des formats XML et JSON. (séance 5 et séance 6)
  • TD 5 : Mise en œuvre d'exemples en utilisant la bibliothèque jQuery. (séance 7)

 

Evaluation

L'évaluation du module est basée sur 3 notes :

  • 1 note de contrôle continu (2*20 minutes en séance)
  • 1 note de TD (assiduité, particpation, rendu)
  • 1 note de DS (1h30)
